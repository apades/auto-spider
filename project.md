核心需求回顾

- 数据源：海南空气质量接口（按站点、年/月/日/小时）。
- 频率：每小时抓取一次（每小时保存当小时和历史数据）。
- 存储与展示：通过 GitHub 静态资源（仓库）提供数据给前端网页读取并显示。
- 自动化：定时爬虫、处理、自动提交到 Git（GitHub），并发布静态页面（GitHub Pages）。
- 部署和运行：使用github action

总体架构（高层）

1. 通过`example/fetch.js`的接口请求样例、和`example/res.json`的返回数据样例，编写一个js爬虫脚本
2. 把爬虫获取到的数据存储在这个仓库中，结构为`/data/$站点/$年/$月/$日/$小时.json`，同时在该目录下生成`all.json`汇总当天 24 小时数据，供前端单次请求
3. 使用github action定时触发，且随时push最新数据到线上github仓库
4. 使用github pages部署静态网页，并通过`https://raw.githubusercontent.com/用户名/仓库名/分支/路径`访问到对应的json数据，放到静态网页中显示
5. 静态网页需要可以切换`$站点/$年/$月/$日`来显示当天所有的数据，然后通过ECharts可视化库显示数据

example/res.json的数据说明

- CDMC  站点
- SO2   空气SO2含量
- PM10  空气PM2含量
- O3    空气O3含量
- PM25  空气PM2.5含量
- NO2   空气NO2含量
- CO    空气CO含量
